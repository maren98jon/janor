# Food App Project Docs Pack

This folder contains the full documentation pack to place inside an empty project folder before asking an implementation AI to build the product.

## Recommended usage order

1. Read `PRODUCT_VISION.md` to understand the product.
2. Read `UX_UI_DIRECTION.md` to lock the design direction.
3. Read `ARCHITECTURE.md` to implement the technical foundation.
4. Read `FEATURES_AND_REQUIREMENTS.md` to implement the functional scope.
5. Read `DATA_MODEL.md` to define entities and persistence.
6. Read `USER_FLOWS_AND_SCREENS.md` to build navigation and screens.
7. Read `MVP_ROADMAP.md` to understand priorities.
8. Use `IMPLEMENTATION_PROMPT.md` as the main prompt for the coding AI.

## Product summary

The product is a mobile-first food management app built with Next.js. It combines home inventory management, expiration tracking, recipes, meal planning, shopping lists, and anti-waste suggestions in one coherent experience.[cite:1][web:13][web:15][web:17]

The UX/UI direction should aim for a premium, natural, calm, Apple-inspired experience based on clarity, deference, and depth rather than on flashy visual effects.[web:53][web:54][web:67]
