# MVP Backlog

## Purpose

This file defines the implementation backlog in a practical and execution-friendly order.[cite:7][cite:69]

| ID | Item | Functional meaning | Priority | Notes / pending points |
|---|---|---|---|---|
| MVP-01 | App shell | Create the mobile-first application shell with bottom navigation and global layout | Must | Navigation labels and safe-area handling must be polished |
| MVP-02 | Design system foundation | Create tokens, spacing, typography, surfaces, buttons, inputs, chips, sheets, and list rows | Must | Must look premium and non-generic |
| MVP-03 | Home screen | Build action-oriented home with expiring soon, cook today, shopping, and quick add | Must | Avoid dashboard feel |
| MVP-04 | Inventory list | Build searchable, filterable inventory list with quick actions | Must | Needs strong scanning on mobile |
| MVP-05 | Add item flow | Build quick-add bottom sheet for manual inventory creation | Must | Friction must be extremely low |
| MVP-06 | Item detail | Build detail view with consume, move, freeze, edit, and discard actions | Must | Use context-sensitive primary action |
| MVP-07 | Recipe list | Build recipe browse/filter view with availability-based ranking | Must | Need All / Available / Use First / Favorites |
| MVP-08 | Recipe detail | Build recipe detail with ingredients, steps, and inventory match status | Must | Must support cook and plan flows |
| MVP-09 | Weekly plan | Build weekly meal planning view | Must | Must remain usable on small screens |
| MVP-10 | Shopping list | Build active shopping list with grouped items and quick check interactions | Must | Must be usable while moving/in store |
| MVP-11 | Mock data layer | Create realistic mock data for items, recipes, plan, and shopping | Must | Data should feel believable |
| MVP-12 | Empty/loading/error states | Build realistic system states for all major screens | Must | No screen should feel unfinished |
| MVP-13 | PWA baseline | Add manifest, icons, installability, and app shell behavior | Should | Offline improvements can be phased |
| MVP-14 | Leftovers | Add leftovers as a basic modeled state or record | Should | Can be simple in first pass |
| MVP-15 | Smarter recipe matching | Improve ranking by expiration urgency and coverage | Should | May ship after baseline matching |
