# Copywriting Guide

## Purpose

This file defines the voice and microcopy rules for the product. The app must sound natural, precise, warm, and product-real. Text should never sound like marketing filler or generic AI output.[cite:1][cite:32]

## Brand voice

The voice should be:

- Calm
- Helpful
- Precise
- Friendly
- Reassuring
- Human

The voice should not be:

- Overexcited
- Corporate
- Cute
- Overly clever
- Robotic
- Vague

## Writing rules

- Prefer short, direct sentences.
- Say exactly what happened.
- Say exactly what the user can do next.
- Avoid abstract product slogans inside the app.
- Use natural household language.
- Use action labels that are immediately understandable.

## Good examples

- Add item
- Use soon
- Added to shopping list
- 3 items expire this week
- Nothing expiring today
- Cook with what you have
- Missing 2 ingredients

## Bad examples

- Optimize your kitchen journey
- Unlock smarter meal experiences
- Maximize household efficiency
- Your pantry, reimagined
- Oops! Something went wrong

## Empty state writing

Empty states should be useful, not decorative.

Pattern:
- What this area is for
- Why it is empty
- What to do next

Example:
- No items in the fridge yet
- Add what you already have at home to start tracking freshness
- Add first item

## Error writing

Error messages should:
- Describe the problem clearly
- Avoid technical blame language
- Offer recovery when possible

Example:
- Couldn’t save this item
- Check your connection and try again

## Success writing

Success feedback should be subtle and short.

Examples:
- Item added
- Plan updated
- Shopping list created
- Recipe saved

## Confirmation writing

Use confirmation only when the action matters.

Examples:
- Discard this item?
- Mark as consumed?
- Remove from this week’s plan?

## Naming style

Use sentence case in the interface. Avoid title case everywhere unless platform conventions clearly require it.

## Food and date language

Use plain real-world terms:

- Expiring soon
- Best before
- Opened
- Frozen
- Leftovers
- Buy again
- Use first

## Final check

Before accepting any UI text, ask:

- Would a real product writer approve this?
- Is it specific?
- Does it sound human?
- Could a tired user understand it instantly?
