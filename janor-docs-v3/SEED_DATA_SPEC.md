# Seed Data Specification

## Purpose

This file defines the quality standard for mock or seed data used in the MVP. Seed data must make the product feel real during implementation and review.

## General rules

- Data must look like real household data, not lorem ipsum.
- Mix short and long names.
- Include partially messy cases.
- Include different expiration states.
- Include different storage locations.
- Include recipes with full and partial coverage.
- Include at least one active shopping list.
- Include at least one week plan with realistic gaps.

## Inventory examples to include

- Greek yogurt
- Eggs
- Spinach bag
- Parmesan block
- Milk
- Tomatoes
- Pasta
- Rice
- Frozen peas
- Chicken breast
- Leftover lentil stew

## Expiration spread

- 2 items expiring today or tomorrow
- 3 items expiring this week
- Several items with no expiration
- At least one expired item for error-state realism

## Recipe examples to include

- Pasta with spinach and parmesan
- Chicken rice bowl
- Yogurt fruit breakfast bowl
- Lentil stew
- Tomato omelette

## Planning examples to include

- A partially filled weekly plan
- One dinner slot left empty
- One plan entry connected to a recipe with missing ingredients

## Shopping examples to include

- Manual item
- Item generated from recipe gap
- Item generated from meal plan
- Mixed categories such as produce, dairy, pantry, frozen
